ORDER BLOCK ⓘ
An Order Block (OB) is a zone where large buyers or sellers placed big orders before a strong move. Price often returns to these zones. This section controls how the screener detects and filters Order Block setups.
How close (in %) price must be to an Order Block zone to be flagged.
How close (in %) price must be to an OB before flagged as approaching. Lower = stricter. 2% is a good default.
How many consecutive candles must stay near a zone to count as consolidation. Try 3–6.
When enabled, only Order Blocks with a strength score above the Min % are kept. Strength is based on how explosive the move was that created the block. Higher min % removes weaker zones and keeps only high-quality blocks. Try 70% for a strong filter.
Near-discretionary OB filter. When enabled, each Order Block gets a quality score using displacement strength, same-direction FVG relationship, liquidity sweep proximity, HTF bias alignment, opposing-zone danger, and freshness/retest behavior. The typed Min Quality value controls how strict the filter is. Higher = fewer but cleaner OBs.
When enabled, the OB approach distance is calculated as (ATR × multiplier) / price instead of a fixed %. Slow/low-volatility coins get a tighter zone; fast/high-volatility coins get a wider zone automatically. 0.5 is a balanced default. Higher = wider approach zone.
FVG ⓘ
A Fair Value Gap (FVG) is a price gap left behind when the market moves very quickly, leaving an area that was not properly traded through. Price often returns to fill these gaps. Use these filters to control which FVGs the screener includes.
When enabled, only FVGs that pass the script's internal quality check are included. Weak or marginal gaps are removed. Keep this on to avoid noisy results — it filters out FVGs that are less likely to act as real price magnets.
Fresh — just formed on recent candles. Untouched — price has not revisited the gap yet (highest probability). Once Touched — price dipped into it once. Old Untouched — older unfilled gap, still valid. Active Retested — currently being revisited by price. Best for entries: Untouched or Fresh.
Filters by how many candles ago the FVG was formed. Min = earliest age allowed, Max = latest. For example Min 0 / Max 5 keeps only FVGs formed in the last 5 candles — very fresh. Use wider ranges to include older unfilled gaps.
Only include FVGs within this percentage distance from current price. Lower value = only show FVGs that price is about to reach. Higher = show gaps further away. Useful for filtering out FVGs that are too far to be actionable right now.
Filters by how deeply price has already entered the gap. Untouched = price has never revisited (cleanest). Edge = only a shallow wick entry. Mid = price entered about halfway. Deep = price filled most of the gap. Use Untouched or Edge for the best setups.
When enabled, only FVGs that overlap with an Order Block are kept. This is a strong confluence filter — areas where both an FVG and an OB align are high-probability zones. Min % sets how much they must overlap. Same Direction means the OB and FVG must both be bullish or both bearish.
FIBONACCI ⓘ
Fibonacci retracement helps find price levels where a pullback is likely to stop and reverse. The screener draws a Fib from a recent swing high to a swing low (or vice versa) and checks if price is sitting on key levels like 0.618 or 0.786.
Enables or disables the Fibonacci module entirely. When off, no Fib calculations are done and no Fib badges will appear in scan results. Turn on to add Fibonacci confluence to your scan.
The timeframe used to calculate the Fibonacci leg. You can set this independently from the main scan timeframe. For example: scan on 15m but use 4h Fibs for higher-quality levels. Higher TF Fibs are generally more reliable.
How close price must be to a Fib level to count as a REACTION hit. 0.5% is strict — price must be almost exactly at the level. Increase if you are getting too few Fib results.
Minimum swing size to qualify as a valid Fib leg, measured in ATR (Average True Range). 1.5 means the swing must be at least 1.5x the ATR. Higher = stricter, only large swings qualify. Lower = more legs found but may include noise. 1.5 is recommended.
Distance from a Fib level at which the APPROACH alert fires. 2.0% means you get alerted when price is within 2% of a level but hasn't touched it yet. The REACTION alert uses Tolerance % instead.
FIB LEVELS ⓘ
Choose which Fibonacci retracement levels to watch. 0.5 = 50% pullback. 0.618 = the "golden ratio" — the most watched level by traders. 0.705 = slightly deeper, often creates strong entries. 0.786 = deep retracement, close to the full swing. Tap a chip to toggle a level on or off.
0.5 0.618 0.705 0.786
When enabled, a Fib hit is only counted if there is also a Fair Value Gap in the same direction near that Fib level. Bullish Fib requires a bullish FVG. Bearish Fib requires a bearish FVG.
When enabled, a Fib hit is only counted if there is also an Order Block in the same direction near that Fib level. Bullish Fib requires a bullish OB. Triple confluence (Fib + OB + FVG) creates very high-probability zones.
BTC CORRELATION ⓘ
BTC Correlation measures how closely a coin's price moves with Bitcoin. A high correlation means the pair mostly follows BTC. A low correlation means it moves more independently. Use this to find pairs that trade on their own momentum instead of just following the market.
Enables the BTC correlation filter. When on, pairs are filtered by how much they move with BTC based on the Mode below. When off, all pairs are scanned regardless of their BTC correlation.
All Scan — no filtering, scan every pair. BTC Correlated — only scan pairs that move closely with BTC (useful during BTC-driven markets). Non-Correlated — only scan pairs moving independently (may have their own strong setups even when BTC is flat).
Number of candles used to calculate the correlation between the pair and BTC. Higher = more stable and smoother reading. Lower = more reactive to recent changes. 60 candles (60 hours on 1h) is a solid default.

True ATH & ATL use full 1D history since listing. Window only controls status detection.
Choose what to scan for. ATH Only = pairs near or at their All-Time High. ATL Only = pairs near or at their All-Time Low. Both = show either. ATH can signal extreme strength or exhaustion. ATL can signal capitulation or a historic buy zone.
At Level Now — price is currently touching the true ATH or ATL (within the Near % threshold). Hit Within Window — the true ATH or ATL was touched recently within the time window you set. Near Level — price is close to the level but may not have touched it yet.
This does NOT recalculate the ATH or ATL — those always use full history since listing. The window only controls how far back to look when checking if the level was recently hit. Only used with "Hit Within Window" status.
How close (in %) current price must be to the true ATH or ATL to be included. 1% = very close to the level. 3–5% = near the level with some room. Higher values will find more results. Used for "At Level Now" and "Near Level" status filters.
